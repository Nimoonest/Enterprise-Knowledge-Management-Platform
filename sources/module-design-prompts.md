# 效果度量与知识溯源模块设计提示词

## 效果度量模块设计提示词

你是企业知识库 / RAG 平台的产品设计顾问，请为“潘婷品牌知识库 Demo”设计一个“效果度量”模块。该模块服务于业务负责人、知识运营和技术同学，需要把 AI 导购、AI Chatbot、GEO 内容检验三个应用的运行效果转化为可汇报、可排查、可优化的指标看板。

请输出一个可直接落地到前端 Demo 的模块方案，至少包含：

- 一级指标：会话量、有效回答率、知识命中率、用户有帮助率、平均质量分、平均响应耗时、Token / 成本等级、风险拦截次数。
- 应用对比：分别展示 AI 导购模拟、AI Chatbot 模拟、GEO 内容检验的指标差异、趋势和主要问题。
- 质量解释：把指标拆成“业务效果、回答质量、检索质量、成本性能、安全合规”五类，并说明每类指标如何判断好坏。
- 行动建议：根据指标自动生成知识补齐、Prompt 优化、测试集扩展、风险词复核、召回阈值调整等建议。
- 页面结构：顶部 KPI 卡片，中部应用对比表，右侧选中应用详情，下方指标口径说明。
- 设计约束：界面应像企业运营控制台，信息密度高、层级清楚、避免营销风格；所有内容必须能用当前潘婷知识库 Demo 的静态样本或真实 MaxKB 调用结果解释。

请用中文输出：模块定位、核心指标、数据结构建议、页面布局、交互逻辑、默认样例数据和后续可接入真实埋点的字段。

## 知识溯源模块设计提示词

你是 RAG 可解释性、审计和知识治理专家，请为“潘婷品牌知识库 Demo”设计一个“知识溯源”模块。该模块需要让演示对象清楚看到：用户问题如何进入某个应用，如何触发检索，命中了哪些知识段落，最终回答是否有依据，以及是否触发风险边界。

请输出一个可直接落地到前端 Demo 的模块方案，至少包含：

- 溯源链路：用户问题 -> 应用入口 -> Prompt / 检索配置 -> 命中段落 -> 回答生成 -> 风险边界 -> 源文件。
- 样本覆盖：出油但发尾干、头屑较多、头皮敏感、GEO JSON 摘要、医疗越界五类典型问题。
- 证据展示：段落标题、知识类型、来源文件、相似度 / Top 分、摘录内容、是否被用于回答。
- 审计视角：展示“已命中、弱命中、风险命中、知识缺口”状态，并给出人工复核动作。
- 页面结构：顶部溯源概览指标，左侧样本列表，右侧链路时间线和证据卡片，下方源文件映射。
- 设计约束：不要展示用户侧不可见的冗长 Prompt；重点展示可解释证据、命中质量和业务可读结论；适配潘婷产品矩阵、导购规则、FAQ、GEO 字段、内容边界、违禁词管理等现有知识模块。

请用中文输出：模块定位、链路字段、证据卡字段、审计状态规则、页面布局、交互逻辑、默认样例数据和后续接入 MaxKB trace / retrieval API 的字段。
